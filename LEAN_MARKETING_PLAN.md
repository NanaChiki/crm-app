### Day 1（今日）: 叔父の提案を深掘り + 実装開始

#### タスク1: 叔父に電話・LINE（15分）

**質問すること:**
```
1. 「年ごとの物件リスト」について：
   - どんな画面・レイアウトをイメージしていますか？
   - どのタイミングで見たいですか？（毎日？週1？月初？）
   - これがあると、具体的にどんな作業が楽になりますか？

2. 現状の使い方について：
   - 今、週に何回アプリを開いていますか？
   - 一番よく使う機能はどれですか？
   - 使いにくい・分かりにくい部分はありますか？

3. お金について（重要！）：
   - もし「年ごとの物件リスト」機能が完璧に実装されたら、
     このアプリにお金を払いますか？
   - いくらまでなら払いますか？
     - 買い切り5,000円？10,000円？30,000円？
     - 月額500円？1,000円？
   - 同業者の友人に勧めますか？
```

**目的:**
- 具体的な実装イメージを掴む
- 「お金を払うかどうか」の本音を聞く
- 同業者紹介の可能性を探る

叔父の回答を基に、実装すべき機能を定義：

**想定される「年ごとの物件リスト」機能:**
```

- 年を選択（2025, 2026, 2027...）

- CSV出力
- 印刷機能
```

#### タスク3: 実装開始（2-3時間）

**優先度1（必須）:**
- 年ごとのメンテナンス予定リストページ作成
- 工事種別ごとにグループ化
- 顧客名・施工年・次回メンテナンス年を表示

**優先度2（あれば良い）:**
- 年選択機能
- フィルタ機能
- 緊急度の色分け

**優先度3（後回し）:**
- CSV出力
- 印刷機能

**実装時間見積もり:**
- Day 1: 2-3時間（基本レイアウト + データ取得ロジック）
- Day 2: 2-3時間（表示ロジック + スタイリング）
- Day 3: 1-2時間（バグ修正 + 微調整）

## 💡 よくある質問

### Q1: 叔父が「お金を払わない」と言ったらどうする？

**A:** まず理由を深掘りする。

```
「なぜお金を払いたくないですか？」
- 機能が足りない → 何が足りない？実装する
- 価格が高い → いくらなら払う？
- 必要性を感じない → 週何回使ってる？使ってないなら製品改善

使っていない = PMF未達成
→ さらなる改善が必要
```

### Q2: ユーザーが全然集まらないんですが...

**A:** アプローチ方法を変える。

```
Twitter効果なし → Facebook広告試す
広告効果なし → ダイレクトアウトリーチ
誰も反応しない → ターゲットが間違っている可能性
```

**代替案:**
- ココナラで建築業経営者にお金を払ってインタビュー
- 叔父経由で直接紹介してもらう
- 建築業のイベント・交流会に参加
- LinkedInで建築業経営者に片っ端から連絡

### Q3: 有料化したら誰も買わないんじゃないか不安...

**A:** だからこそ、事前に「お金を払うか？」を確認する。

```
Week 1-4で確認すべきこと:
「これにお金を払いますか？」
「いくらまでなら払いますか？」

3人以上が「払う」と言ったら → 有料化OK
誰も「払う」と言わない → PMF未達成、改善必要
```

### Q4: 買い切りと月額、どっちがいい？

**A:** ユーザーに聞く。一般的には：

```
買い切りのメリット:
- ユーザーは月額を嫌う（特に50代）
- 一度の売上が大きい
- キャッシュフロー改善

月額のメリット:
- 継続的な収入（MRR）
- LTVが高い
- 価格が低く見える

推奨: 両方提供して、ユーザーに選ばせる
```
